项目：访达右键增强工具
版本：V1.0.0

你正在实现一个 macOS 访达右键增强工具，请严格遵守以下要求：

1. 所有界面文案使用中文。
2. 项目由主应用和 Finder 扩展两部分组成。
3. 配置必须由共享模型驱动，并通过 App Group 在主应用与扩展之间共享。
4. MVP 仅实现：
   - 新建文件夹
   - 新建 txt / md / json
   - 在终端中打开
   - 用 VS Code 打开
   - 复制路径
   - 菜单配置页
5. 所有新增功能必须补充测试或更新测试清单。
6. 完成功能后，输出修改文件、测试命令、测试结果、建议版本号和是否需要更新 CHANGELOG。
7. 不要擅自引入数据库、网络服务、插件系统。
8. 代码结构必须遵循设计文档中的模块划分。
9. 如修改共享配置模型，必须同步更新文档与回归测试用例。
10. 提交信息遵循以下格式：
    - feat: 新功能
    - fix: 修复
    - refactor: 重构
    - test: 测试
    - docs: 文档
    - chore: 工程杂项

推荐实施顺序：
1. 建立 Xcode 工程与 target 结构。
2. 实现共享模型与 App Group 存储。
3. 实现主应用中的菜单管理页面。
4. 实现 Finder 扩展读取配置并展示静态菜单。
5. 打通第一个动作：在终端中打开。
6. 实现新建文件夹。
7. 实现新建 txt / md / json。
8. 实现复制路径。
9. 实现 VS Code 打开逻辑。
10. 增加测试、回归清单与文档版本更新。

每次提交前请输出：
- 本次修改的文件列表
- 本次新增或修改的测试
- 已执行的测试命令
- 测试结果
- 是否影响回归用例 1-7
- 是否需要更新文档版本号与 CHANGELOG
