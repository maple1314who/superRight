## 一、执行要求

每次功能开发完成后，至少完成以下检查：

- 单元测试全量通过
- 核心回归用例 1-7 验证通过
- 如修改共享模型，文档与测试夹具已同步更新
- `CHANGELOG.md` 已检查是否需要更新

## 二、核心回归用例

### 用例 1：空白处右键菜单过滤
- 前置条件：启用“新建文件夹”“新建 Markdown 文件”“复制路径”
- 操作：在任意文件夹空白处右键
- 预期：显示上述菜单项；不显示仅针对文件场景的菜单项

### 用例 2：文件右键菜单过滤
- 前置条件：启用“用 VS Code 打开”“复制路径”，关闭“新建文件夹”
- 操作：选中一个文件右键
- 预期：显示 VS Code 与复制路径；不显示新建文件夹

### 用例 3：文件夹右键打开终端
- 前置条件：启用“在终端中打开”
- 操作：选中文件夹后右键点击“在终端中打开”
- 预期：Terminal 在该文件夹路径打开

### 用例 4：新建 Markdown 文件
- 前置条件：启用“新建 Markdown 文件”
- 操作：空白处右键点击该菜单
- 预期：在当前目录创建 `Untitled.md`，内容为 `# 标题`

### 用例 5：命名冲突处理
- 前置条件：当前目录已存在 `Untitled.md`
- 操作：再次执行“新建 Markdown 文件”
- 预期：创建 `Untitled 2.md`

### 用例 6：配置变更即时生效
- 前置条件：在主应用中关闭“复制路径”
- 操作：回到 Finder 右键
- 预期：菜单中不再展示“复制路径”

### 用例 7：未安装应用隐藏
- 前置条件：未安装 Cursor，且开启“隐藏未安装应用菜单”
- 操作：打开菜单管理页与 Finder 菜单
- 预期：不展示 Cursor 相关菜单
