## 2. 每个文件改动说明

- `SharedConfiguration.swift`
  - 默认菜单项收敛到 V2 指定的 5 个动作：
    - 新建文件夹
    - 在终端中打开
    - 在 iTerm 中打开
    - 用 VS Code 打开
    - 复制路径
  - 保留现有共享模型结构，未重构数据模型。

- `MenuManagementViewModel.swift`
  - 在开关、可见性、排序、应用设置、应用路径变更后自动调用 `store.save(...)`。
  - 保留原有 `save()` 显式保存能力。

- `FinderSelectionContext.swift`
  - 新增 `fromFinder(targetedURL:selectedItemURLs:)`。
  - 新增 `resolveCurrentDirectoryURL(...)`，并按 V2 规则处理：
    - 右键文件夹：当前目录为该文件夹
    - 右键文件：当前目录为文件父目录
    - 空白右键：当前目录为当前目录（targetedURL）

- `FinderSync.swift`（Xcode 工程）
  - 从测试菜单切换为正式桥接实现：
    1. 从 Finder 获取 `targetedURL + selectedItemURLs`
    2. 构造 `FinderSelectionContext`
    3. 读取 `SharedConfiguration`
    4. 调用 `MenuBuilder.buildMenu(...)`
    5. 动态生成 `NSMenuItem`
    6. 点击后统一走 `ActionDispatcher.execute(...)`
  - `representedObject` 使用 `NSObject` 包装 payload。
  - 全部关键日志使用 `NSLog`。

- `FinderSelectionContextTests.swift`
  - 新增目录推导单测（文件夹/文件/空白）。

- `MenuBuilderTests.swift`
  - 更新为 V2 5 动作菜单断言。
  - 新增未安装应用隐藏逻辑测试。

- `ActionDispatcherTests.swift`
  - 调整为 V2 结构下可执行的创建文件冲突测试（通过自定义 `create_file` item）。

- `MenuManagementViewModelTests.swift`
  - 新增自动保存触发测试，确保 UI 改动不只停留在内存态。

## 6. 当前已知问题与风险

- 风险 1：`UserDefaultsConfigurationStore` 当前依赖 `suiteName = group.com.superright.shared`。若主 App 与 Extension 的 App Group Entitlement 未在 Xcode 中一致配置，运行时可能出现“各读各的”或读取失败。
- 风险 2：Finder Extension 监听目录当前仍是初始化写死的 `/Users/maple/Desktop`。如要覆盖更多目录，建议改为可配置或按产品策略扩展监听范围。
- 风险 3：复制路径使用 `NSPasteboard`，在某些系统权限/上下文限制下可能存在剪贴板写入失败边缘情况（代码已输出失败日志）。

## 7. 后续建议

1. 在 Xcode 中核对主 App 与 Extension 的 App Group Entitlement 一致性（必须同组）。
2. 按 `验证操作文档_V2.0.0` 完整执行 Finder 手工联调并记录截图/日志。
3. 若后续需要沙盒更稳的跨进程文件访问，可评估安全书签（Security-Scoped Bookmark）方案。
