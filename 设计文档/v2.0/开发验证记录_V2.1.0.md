## 2. 具体修复内容

### 2.1 P0-1 配置不同步修复

1. App Group entitlement 对齐
- 主 App entitlement：`右键增强/右键增强.entitlements`
- Extension entitlement：`RightClickFinderExtension/RightClickFinderExtension.entitlements`
- 两者一致：
  - `com.apple.security.app-sandbox = true`
  - `com.apple.security.application-groups = ["group.com.superright.shared"]`

2. Build Settings 显式绑定 entitlement
- 主 App target：`CODE_SIGN_ENTITLEMENTS = 右键增强/右键增强.entitlements`
- Extension target：`CODE_SIGN_ENTITLEMENTS = RightClickFinderExtension/RightClickFinderExtension.entitlements`

3. Store 可观测化
- `UserDefaultsConfigurationStore` 新增可读字段：
  - `suiteName`
  - `key`
- 新增 `debugSummary(prefix:)`，用于日志输出当前存储上下文。

4. 主 App 侧配置日志
- `MenuManagementViewModel` 在 `init/load/save/autosave` 输出：
  - 实际 suiteName
  - menuItems.count
  - 每个 menu item 的 `id=on/off`

### 2.2 P0-2 点击无效修复

1. FinderSync 链路日志补齐（NSLog）
- `menu(for:)`：调用日志、targetedURL、selectedURLs、scene、currentDirectory、配置快照
- `buildNSMenu`：displayItems 数量
- `handleMenuItem`：是否触发、sender 解包、payload 解包、点击动作信息
- `dispatcher.execute()` 前后：调用点与成功/失败日志

2. FinderSync 结构收敛
- 新增 `buildNSMenu(displayItems:context:)`，避免 `menu(for:)` 内堆业务细节
- 点击方法统一 `handleMenuItem(_ sender: Any?)`，增强兼容与日志可观测性

3. ActionDispatcher 增加执行级日志
- `execute()` 入口、动作分支结果、open 命令参数与状态码全部记录

## 9. 下一步建议（针对你当前本机联调）

1. 在 Xcode 里分别 Run 主 App 与 Finder Extension（附着 Finder）。
2. 观察 Finder 扩展日志中的以下关键点是否连续出现：
   - `menu(for:) called`
   - `buildNSMenu called`
   - `handleMenuItem called`
   - `payload decode success`
   - `dispatcher.execute() about to call`
   - `Action executed` 或 `Action failed`
3. 按 A2/A3 场景手工验证“在终端中打开”开关前后菜单变化（你本机 GUI 环境最终确认）。
