## 2. 每个文件改动说明

### 2.1 `右键增强.xcodeproj/project.pbxproj`
- 对齐主 App 与 Extension 的 Debug/Release 签名与能力，避免父子签名不一致：
  - 主 App Debug：
    - `CODE_SIGN_ENTITLEMENTS = 右键增强/右键增强.entitlements`
    - `CODE_SIGN_IDENTITY = Apple Development`
    - `ENABLE_APP_SANDBOX = YES`
    - `REGISTER_APP_GROUPS = YES`
  - 主 App Release：
    - 补齐 `CODE_SIGN_IDENTITY = Apple Development`
  - Extension Debug：
    - `CODE_SIGN_ENTITLEMENTS = RightClickFinderExtension/RightClickFinderExtension.entitlements`
    - `ENABLE_APP_SANDBOX = YES`
- 结果：`ValidateEmbeddedBinary` 可通过，不再报 embedded binary 证书不一致。

### 2.2 `RightClickFinderExtension/FinderSync.swift`
- 保持并加强真实点击链路 `NSLog`：
  - `menu(for:) called`
  - `buildNSMenu called`
  - `handleMenuItem called`
  - `sender title`
  - `representedObject type`
  - `payload decode success/failed`
  - `dispatcher.execute() about to call`
  - `Action executed/Action failed`
- 增加上下文兜底逻辑：
  - 当 `targetedURL == nil` 时，回退 `effectiveTargetedURL = directoryURLs.first`
  - 目的是避免在部分 Finder 场景下上下文为空直接返回 `nil`，导致菜单不显示。

3. 执行：

   - `menu(for:) called`
   - `buildNSMenu called`
5. 点击“新建文件夹”，观察是否出现：
   - `handleMenuItem called`
   - `payload decode success`
   - `dispatcher.execute() about to call`
   - `Action executed`
