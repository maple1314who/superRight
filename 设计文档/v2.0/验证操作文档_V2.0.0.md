# 验证操作文档 V2.0.0

## 1. 文档目的
本文档用于验证 macOS 访达右键增强工具 V2 MVP 是否符合预期。
验证对象包括：
- 主 App 配置界面
- Shared 配置存储
- Finder 右键菜单生成
- ActionDispatcher 动作执行
- 主 App 与 Finder 配置联动

## 2. 验证前提
在开始验证前，请确认：
1. Xcode 主工程路径为：
   `/Users/maple/Documents/Project/Swift/右键增强`
2. Swift Package 路径为：
   `/Users/maple/Documents/Project/Swift/SuperRight`
3. 主 App 与 Finder Extension 均可编译通过
4. Finder Extension 已能运行并可被系统加载
5. Finder 扩展菜单已至少能在桌面或指定目录显示

## 3. 运行准备
### 3.1 运行主 App
- 在 Xcode 中选择主 App Scheme
- 点击 Run
- 确认可以打开配置界面

### 3.2 运行 Finder Extension
- 在 Xcode 中选择 Finder Extension Scheme
- 点击 Run
- 如系统要求附着 Finder，则选择 Finder
- 如必要，执行：`killall Finder`

### 3.3 观察日志
Finder Extension 日志建议通过 `NSLog` 查看。
需要关注以下类型日志：
- 扩展启动日志
- menu(for:) 调用日志
- 当前上下文日志
- 点击菜单日志
- 动作执行成功/失败日志

## 4. 验证范围
V2 MVP 验证范围包括 5 个动作：
1. 新建文件夹
2. 在终端中打开
3. 在 iTerm 中打开
4. 用 VS Code 打开
5. 复制路径

## 5. 主 App 配置验证
### 用例 A1：主 App 可正常打开
步骤：
1. 运行主 App
2. 打开配置页面

预期结果：
- 应显示菜单管理界面
- 不崩溃
- 不出现空白页

### 用例 A2：菜单项开关可修改
步骤：
1. 在主 App 中关闭“在终端中打开”
2. 保存或确认配置已自动保存
3. 回到 Finder 重新打开右键菜单

预期结果：
- Finder 菜单中不再显示“在终端中打开”

### 用例 A3：再次启用菜单项
步骤：
1. 在主 App 中重新开启“在终端中打开”
2. 回到 Finder 重新右键

预期结果：
- Finder 菜单中重新出现“在终端中打开”

### 用例 A4：排序变更生效
步骤：
1. 在主 App 中调整菜单项顺序
2. 回到 Finder 重新打开菜单

预期结果：
- 菜单顺序与配置一致

## 6. Finder 菜单展示验证
### 用例 B1：桌面空白处右键
步骤：
1. 进入桌面
2. 在空白区域右键

预期结果：
- 若当前场景允许显示菜单，则显示对应菜单项
- 菜单项顺序符合配置
- 无明显错乱或重复菜单

### 用例 B2：右键文件夹
步骤：
1. 在桌面选择一个文件夹
2. 右键该文件夹

预期结果：
- 菜单按“文件夹场景”显示
- 菜单项与主 App 配置一致

### 用例 B3：右键文件
步骤：
1. 在桌面选择一个普通文件
2. 右键该文件

预期结果：
- 菜单按“文件场景”显示
- 菜单项与主 App 配置一致

## 7. 动作执行验证
### 用例 C1：新建文件夹
步骤：
1. 在桌面空白处或某个目录内右键
2. 点击“新建文件夹”

预期结果：
- 在当前目录创建一个新文件夹
- 命名冲突时自动递增（如“新建文件夹 2”）
- 不应创建到错误目录

### 用例 C2：右键文件夹后新建文件夹
步骤：
1. 右键一个文件夹
2. 点击“新建文件夹”

预期结果：
- 新文件夹应创建在该文件夹内（如果产品定义如此）
- 或创建在该文件夹所在当前目录（取决于最终设计）
- 必须与 FinderSelectionContext 的逻辑保持一致，且符合预期

### 用例 C3：在终端中打开
步骤：
1. 在 Finder 中右键一个目录
2. 点击“在终端中打开”

预期结果：
- Terminal 被拉起
- 当前目录正确切换到目标路径
- 控制台日志记录 action 执行成功

### 用例 C4：在 iTerm 中打开
前提：机器已安装 iTerm

步骤：
1. 在 Finder 中右键一个目录
2. 点击“在 iTerm 中打开”

预期结果：
- iTerm 被拉起
- 目标路径正确
- 控制台日志记录 action 执行成功

补充：
若未安装 iTerm 且配置要求隐藏未安装应用，则菜单中不应显示该项。

### 用例 C5：用 VS Code 打开
前提：机器已安装 VS Code

步骤：
1. 右键某个文件夹或文件
2. 点击“用 VS Code 打开”

预期结果：
- VS Code 成功打开目标文件或文件夹
- 控制台日志记录 action 执行成功

补充：
若未安装 VS Code 且配置要求隐藏未安装应用，则菜单中不应显示该项。

### 用例 C6：复制路径
步骤：
1. 右键某个文件或文件夹
2. 点击“复制路径”
3. 在任意文本输入框粘贴

预期结果：
- 粘贴结果为正确路径
- 路径与当前目标一致
- 控制台日志记录 action 执行成功

## 8. 配置联动验证
### 用例 D1：关闭菜单项后 Finder 中立即隐藏
步骤：
1. 主 App 中关闭“复制路径”
2. 返回 Finder
3. 重新右键

预期结果：
- Finder 菜单中“复制路径”消失

### 用例 D2：开启菜单项后 Finder 中恢复显示
步骤：
1. 主 App 中开启“复制路径”
2. 返回 Finder
3. 重新右键

预期结果：
- Finder 菜单中“复制路径”恢复显示

### 用例 D3：隐藏未安装应用
前提：配置开启 `hideUnavailableApplications`

步骤：
1. 确保机器未安装某目标 App（如 iTerm 或 VS Code）
2. 回到 Finder 查看菜单

预期结果：
- 未安装应用对应菜单项不显示

### 菜单生成日志

### 动作执行日志
- `Action executed`
- `Action failed`

## 10. 失败排查建议
### 10.1 菜单能显示但点击没效果
优先检查：
- `NSMenuItem.target`
- `NSMenuItem.action`
- `representedObject` 是否为可桥接对象
- 是否使用 `NSObject` 包装 payload
- 是否使用 `NSLog` 输出日志

### 10.2 Finder 菜单不出现
优先检查：
- Finder Extension 是否已运行
- 是否在监听目录内测试
- `menu(for:)` 是否被调用
- Finder 是否已重启

### 10.3 菜单有但内容不对
优先检查：
- `SharedConfiguration` 是否正确读取
- `MenuBuilder.buildMenu(...)` 过滤逻辑
- 当前 `FinderSelectionContext.scene` 是否判断正确

### 10.4 新建文件夹位置不对
优先检查：
- `FinderSelectionContext.currentDirectoryURL`
- 文件 / 文件夹 / 空白区域的 currentDirectoryURL 逻辑是否正确

## 12. V2 验收标准
满足以下条件可判定 V2 MVP 验收通过：
1. 主 App 配置界面可用
2. 配置改动能真正持久化
3. Finder 菜单由配置动态生成
4. Finder 菜单点击能执行真实动作
5. 至少以下 5 项动作已覆盖并验证：
   - 新建文件夹
   - 在终端中打开
   - 在 iTerm 中打开
   - 用 VS Code 打开
   - 复制路径
6. 配置与 Finder 展示具有联动效果
