# 开发验证记录 V3.10.1

日期：2026-06-09

## 目标

修复 full-size content view 下整窗背景可拖拽导致表格行拖拽、Picker 下拉选择被干扰的问题。

## 已完成

- 关闭 `NSWindow.isMovableByWindowBackground`，避免整个内容区都触发拖窗。
- 新增顶部专用拖拽热区，仅覆盖右侧内容区顶部空白区域。
- 顶部拖拽热区避开左侧栏、左上角系统窗口按钮，且不延伸到表格内容区域。
- 保留 V3.10.0 的顶部视觉留白风格。
- 主 App 与 Finder Extension 版本推进到 `3.10.1 / 2026060919`。

## 手动验证建议

1. 在窗口顶部空白区域拖动，确认窗口仍可移动。
2. 在表格行内拖拽排序，确认不会触发拖窗。
3. 打开新建/工具箱等页面的 Picker 或菜单，确认能正常点击选择。
4. 点击左上角红黄绿窗口按钮，确认不会被拖拽热区遮挡。

- `launchAtLogin` 仍只有配置字段，尚未接入 Login Item。
- Developer ID 签名与公证仍受证书和 `notarytool` 凭据限制。
