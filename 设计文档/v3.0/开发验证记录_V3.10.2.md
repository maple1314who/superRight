# 开发验证记录 V3.10.2

日期：2026-06-09

## 目标

补齐旧版 `iRightMouse` 工具箱 `arrOfOthers` 中的工具入口，让“工具箱”页面和 Finder 右键菜单覆盖旧版截图里的功能项。

## 已完成

- `ToolboxItemConfiguration.defaultItems` 扩展到旧版 46 个工具入口。
- 新增工具动作类型：快捷方式到桌面、macOS/iOS 图标集、外部 App 打开、翻译、二维码。
- Finder Extension 新增对应转发分支，继续通过 App Group 队列和分布式通知交给主 App 执行。
- 主 App 新增真实执行逻辑：
  - 在桌面创建选中项符号链接。
  - 生成 `.icns`、`.iconset`、`AppIcon.appiconset`。
  - 通过 App 名称或 Bundle ID 打开选中项。
  - 用选中项名称打开百度/Google 翻译。
  - 将选中项路径生成二维码图片写入剪贴板。
- 主 App 与 Finder Extension 版本推进到 `3.10.2 / 2026060920`。

## 手动验证建议

1. 启动 `右键增强.app`，进入“工具箱”，确认旧版功能入口已补齐。
2. 在 Finder 选中文件右键，确认“工具”子菜单出现已启用工具项。
3. 验证“发送快捷方式到桌面”“生成二维码”“生成 macOS Icons”“生成 iOS Icons”。
4. 启用某个外部 App 入口，确认已安装时能打开选中项，未安装时记录失败日志。
5. 选中图片执行“ICNS 转换”，确认同目录生成 `.icns`。

## 已知限制

- iShot 的“标注/贴图”和“截图”在 V4.0.30 已改为区分执行：标注优先用 iShot 打开选中项，截图优先触发默认 `Option+A` 快捷键；未授权辅助功能时回退启动主应用。
- 外部 App 入口依赖本机安装路径或 Bundle ID 解析；冷门 App 未安装时只记录失败，不会弹出额外错误 UI。
- “发送快捷方式到桌面”当前使用符号链接实现，不是 Finder 原生 alias 文件。
