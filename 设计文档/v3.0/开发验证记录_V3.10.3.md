# 开发验证记录 V3.10.3

日期：2026-06-09

## 目标

修正 V3.10.1 只允许顶部拖窗的交互偏差：窗口应保持大面积背景可拖拽，但表格、Picker、文本输入等内部交互不能被外层拖窗抢占。

## 已完成

- 主窗口改为自定义全窗背景拖拽覆盖层。
- 保持 `NSWindow.isMovableByWindowBackground = false`，避免系统背景拖窗无差别抢占 SwiftUI 拖拽。
- 新增 `WindowDragExclusionMarkerView`，由表格容器写入 AppKit 标记。
- 覆盖层命中表格排除标记、AppKit 控件、滚动视图、文本控件、Picker/Button 类视图时放行事件。
- 左上角系统红黄绿按钮区域直接放行，不被拖窗覆盖层遮挡。
- 主 App 与 Finder Extension 版本推进到 `3.10.3 / 2026060921`。

## 手动验证建议

1. 在窗口背景空白区域拖动，确认窗口可移动。
2. 在“发送文件到...”“常用目录”“文件(夹)图标”“新建文件”的表格行内拖拽排序，确认不触发拖窗。
3. 在“工具箱”页面点击 Picker/文本项/复选框，确认不会被拖窗覆盖层拦截。
4. 点击左上角红黄绿窗口按钮，确认系统按钮仍可用。

## 已知限制

- 全窗拖拽覆盖层依赖 AppKit 命中判断和显式排除标记；如后续新增复杂自定义 SwiftUI 控件，需要同步加 `WindowDragExclusionMarker`。
