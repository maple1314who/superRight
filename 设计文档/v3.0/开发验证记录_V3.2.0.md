# 开发验证记录 V3.2.0

- 日期：2026-06-09
- 执行人：Codex
- 版本目标：把“常用目录”从 UI 占位改为真实配置和右键可执行动作。

## 1. 本次完成范围

### 1.1 配置模型
- 复用 `FileDestinationConfiguration` 表达常用目录。
- `SharedConfiguration` 新增 `favoriteDirectories`。
- `AppSettings` 新增：
  - `showFavoriteDirectoryIcons`
  - `enableFavoriteDirectories`
- 旧配置解码时自动补默认常用目录。

### 1.2 设置界面
- “常用目录”设置页不再使用本地假数据。
- 支持：
  - 新增目录
  - 删除目录
  - 重置默认目录
  - 编辑真实路径
  - 编辑显示名称
  - 显示图标开关
  - 启用常用目录开关

### 1.3 右键菜单
- Finder 菜单构建时读取 `favoriteDirectories`。
- 当“启用常用目录”开启时，在空白处、文件、文件夹场景都生成：
  - `打开 音乐`
  - `打开 图片`
  - `打开 影片`
  - 用户自定义目录

### 1.4 执行动作
- 新增 `MenuActionType.openDirectory`。
- 新增 `AppExecutionAction.openDirectory`。
- Finder Extension 点击后转发主 App。
- 主 App 校验目标目录存在后使用 `NSWorkspace.shared.open` 打开目录。
- ExtensionCore 中保留 `/usr/bin/open` 执行能力，便于单测和后续边界调整。

## 4. 手工验证清单

### 用例 A：默认常用目录显示
步骤：
1. 打开 App。
2. 进入“常用目录”。
3. 确认默认目录：音乐、图片、影片。

预期：
- 表格读取真实配置。
- 修改路径或名称后，重新打开 App 仍保留。

### 用例 B：右键打开常用目录
步骤：
1. 确认“启用常用目录”已勾选。
2. 在监听目录空白处或选中文件后右键。
3. 点击“打开 图片”。

预期：
- Finder 打开图片目录。
- 主 App 日志出现 `openDirectory started` 和 `openDirectory success`。

### 用例 C：禁用常用目录
步骤：
1. 取消勾选“启用常用目录”。
2. 回到 Finder 重新右键。

预期：
- 不再显示 `打开 <目录名>` 菜单项。
