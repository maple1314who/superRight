## 1. 本次完成范围

### 1.1 共享配置
- 新增 `FileIconConfiguration`，保存文件/文件夹图标预设的启用状态、显示名称、排序、SF Symbol 和颜色。
- `SharedConfiguration` 新增 `fileIconPresets`，旧配置缺失该字段时自动补默认预设。
- `AppSettings` 新增：
  - `showFileIconPresetIcons`：控制配置界面是否显示预设图标。
  - `enableFileIconPresets`：控制 Finder 右键菜单是否生成图标动作。
- `AppExecutionRequest` 新增图标动作和字段：
  - `applyFileIcon`
  - `removeCustomIcon`
  - `iconSystemImageName`
  - `iconColorName`

### 1.2 配置界面
- “文件/文件夹图标”页面改为绑定真实共享配置。
- 支持启用/禁用预设、编辑显示名称、新增预设、删除最后一个预设、恢复默认预设。
- 支持配置开关“显示图标”和“在 Finder 右键菜单启用图标预设”。

### 1.3 Finder 右键菜单
- 选中文件或文件夹时，根据启用的图标预设生成“设置图标：<名称>”。
- 选中文件或文件夹时生成“还原默认图标”。
- 空白处右键不显示图标动作。
- Finder Extension 将图标动作转发给主 App，并携带选中路径、SF Symbol 和颜色字段。

### 1.4 主 App 执行
- 主 App 支持执行 `applyFileIcon`，使用 SF Symbol 和颜色生成 `128 x 128` 自定义图标。
- 主 App 支持执行 `removeCustomIcon`，通过 `NSWorkspace.setIcon(nil, forFile:)` 恢复默认图标。
- 图标写入在主线程执行，减少 AppKit 调用的不确定性。
- 请求队列日志文案统一为通用执行失败信息，不再只写 createFolder/createFile。

## 4. 手工验证建议

1. 启动 `右键增强.app`，进入“文件/文件夹图标”。
2. 确认默认预设列表可见，开关、名称编辑、新增、删除、恢复默认可保存。
3. 在 Finder 中选中文件或文件夹右键，确认出现“设置图标：...”和“还原默认图标”。
4. 点击一个“设置图标”动作，确认目标文件或文件夹 Finder 图标发生变化。
5. 点击“还原默认图标”，确认自定义图标被移除。
6. 如果菜单未刷新，重启 Finder Extension 或重新打开 Finder 窗口后再验证。
