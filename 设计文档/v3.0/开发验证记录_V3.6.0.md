# 开发验证记录 V3.6.0

- 日期：2026-06-09
- 执行人：Codex

- 版本号：`3.6.0 / 2026060906`

## 1. 本次完成范围

### 1.1 共享配置
- `NewFileTemplateConfiguration` 新增 `templateData`，保存从真实模板文件读取的原始字节。
- `MenuItemConfiguration` 和 `MenuDisplayItem` 新增 `templateData`，确保菜单构建链路不丢失模板数据。
- `AppExecutionRequest` 新增可选 `templateData` 字段，并保持旧 JSON 可解码。

### 1.2 配置界面
- “添加模板文件”改为打开 `NSOpenPanel`，选择真实文件后导入为新建文件模板。
- 新增“添加空白模板”，保留原来的手动模板新增方式。
- 导入失败时在当前设置页显示错误信息。
- 导入模板自动使用文件名、扩展名和图标元数据生成配置。

### 1.3 Finder 与主 App 执行
- `MenuBuilder` 将模板原始数据传入动态 `createFile` 菜单项。
- Finder Extension 转发 `createFile` 请求时携带 `templateData`。
- 主 App 创建文件时优先写入 `templateData`，无原始字节时回退到 `templateContent`。
- ExtensionCore 的参考执行器同步支持原始字节写入，保证单元测试覆盖。

## 4. 手工验证建议

1. 启动 `右键增强.app`，进入“新建文件”。
2. 点击“添加模板文件”，选择一个真实文件模板，例如 `.rtf`、`.docx`、`.json` 或二进制样例。
3. 确认模板列表新增对应条目，标题、扩展名、默认文件名正确。
4. 在 Finder 右键选择该模板创建文件。
5. 对比新建文件和源模板文件内容，确认原始字节一致。
6. 再测试“添加空白模板”，确认旧的手动新增方式仍可用。

## 5. 已知限制

- 模板原始字节会进入 App Group 配置，超大模板会增大配置体积；后续可改为 App Group 文件存储。
- 当前没有模板内容预览和模板重命名专用面板，只提供基础列表配置。
- Finder 右键菜单仍是扁平列表，尚未实现分组/子菜单语义。
