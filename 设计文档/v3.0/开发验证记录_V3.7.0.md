# 开发验证记录 V3.7.0

- 日期：2026-06-09
- 执行人：Codex
- 版本目标：工具箱新增隔空投送动作，并完成共享配置、Finder 菜单和主 App 执行链路。
- 版本号：`3.7.0 / 2026060907`

## 1. 本次完成范围

### 1.1 共享配置
- `MenuActionType` 新增 `sendViaAirDrop`。
- `AppExecutionAction` 新增 `sendViaAirDrop`，并明确该动作不绑定外部应用路径。
- `ToolboxItemConfiguration.defaultItems` 新增“隔空投送”默认项。
- 旧配置通过 `SharedConfiguration.upgradedWithDefaults()` 自动追加缺失的 `send_via_airdrop` 项。

### 1.2 Finder 右键菜单
- `MenuBuilder` 复用工具箱可见性规则，选中文件或文件夹时展示“隔空投送”。
- 空白处场景不展示“隔空投送”，避免没有选中项时触发无效分享。
- `ActionDispatcher` 保持该动作为 forwarded-only，避免 Finder Extension 进程直接调起系统分享 UI。

### 1.3 主 App 执行
- Finder Extension 点击“隔空投送”后转发 `sendViaAirDrop` 请求。
- 主 App 读取 `sourcePaths`，通过 `NSSharingService(named: .sendViaAirDrop)` 调起系统 AirDrop 面板。
- 如果系统分享服务不可用，写入明确错误日志。

## 4. 手工验证建议

1. 启动 `右键增强.app`，进入“工具箱”。
2. 确认工具箱列表中存在“隔空投送”，且可启用、禁用、改名。
3. 在 Finder 选中文件或文件夹右键，点击“隔空投送”。
4. 确认系统 AirDrop 分享面板弹出，并包含选中项。
5. 在 Finder 空白处右键，确认不出现“隔空投送”。

## 5. 已知限制

- AirDrop 发送过程由系统分享服务接管，代码只能确认面板调起，不能自动确认用户最终发送成功。
- 如果当前系统关闭 AirDrop 或分享服务不可用，会记录错误，不做自定义替代实现。
- Finder 右键菜单仍是扁平列表，尚未实现分组/子菜单语义。
