# 开发验证记录 V3.8.0

- 日期：2026-06-09

- 版本号：`3.8.0 / 2026060908`

## 1. 本次完成范围

### 1.1 共享配置
- `MenuActionType` 新增 `permanentlyDelete`。
- `AppExecutionAction` 新增 `permanentlyDelete`，并明确该动作不绑定外部应用路径。
- `ToolboxItemConfiguration.defaultItems` 新增“彻底删除”默认项。
- “彻底删除”默认关闭，旧配置迁移时只追加配置项，不自动进入右键菜单。

### 1.2 Finder 右键菜单
- 用户在工具箱启用“彻底删除”后，选中文件或文件夹时 Finder 右键菜单展示该动作。
- 空白处场景不展示“彻底删除”，避免没有选中项时误触。
- `ActionDispatcher` 保持该动作为 forwarded-only，避免 Finder Extension 进程直接删除文件。

### 1.3 主 App 执行
- Finder Extension 点击“彻底删除”后转发 `permanentlyDelete` 请求。
- 主 App 根据 `sourcePaths` 调用 `FileManager.removeItem(at:)` 直接删除选中项。
- 删除失败时写入路径和错误日志，并返回明确的业务错误。

## 4. 手工验证建议

1. 启动 `右键增强.app`，进入“工具箱”。
2. 确认工具箱列表中存在“彻底删除”，默认关闭。
3. 启用“彻底删除”，在 Finder 选中临时测试文件右键。
4. 点击“彻底删除”，确认文件被直接删除且不进入废纸篓。
5. 在 Finder 空白处右键，确认不出现“彻底删除”。
