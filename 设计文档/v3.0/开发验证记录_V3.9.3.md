# 开发验证记录 V3.9.3

- 日期：2026-06-09
- 执行人：Codex

- 版本号：`3.9.3 / 待发布`

## 1. 已完成范围

### 1.1 配置界面空入口清理
- 删除侧边栏顶部“评分”按钮。
- 删除侧边栏底部“订购”按钮。
- 这两个入口此前没有动作实现，删除后避免用户误以为评分、订阅能力已接入。

### 1.2 Finder 右键分组子菜单
- Finder Extension 继续接收 `MenuBuilder` 输出的平铺 `MenuDisplayItem`。
- 展示层按 `MenuGroup` 构建“新建 / 打开 / 工具”子菜单。
- 叶子菜单项仍通过唯一 tag 映射 `MenuActionPayload`，点击执行和主 App 转发链路不变。

### 1.3 文件/文件夹图标导入与排序
- 图标预设新增 `importedImageData` 和 `importedImageFileName`，旧配置缺字段时仍可正常解码。
- 配置界面每个图标预设支持“导入 / 更换 / 清除”本地图片，并在表格中预览导入图片。
- 图标预设表支持按住行拖拽排序，排序后重写 `order` 并自动保存到共享配置。
- Finder 右键“设置图标”请求会携带导入图片数据，主 App 执行时优先使用导入图片，未导入时回退到内置 SF Symbol。

### 1.4 工具箱扩展项
- 默认工具箱新增“剪切”“iShot”“ICNS 转换”，旧配置加载时会自动追加缺失项。
- “剪切”把 Finder 选中项写入系统剪贴板，并附加 move 语义供后续粘贴移动。
- “iShot”从 bundle identifier 或 `/Applications/iShot.app` 启动本机 iShot。
- “ICNS 转换”把选中图片生成标准 `.iconset` 后调用 `/usr/bin/iconutil` 输出同目录 `.icns` 文件，重名时自动追加序号。

### 1.5 彻底删除二次确认
- “彻底删除”在主 App 执行前弹出 `NSAlert` 二次确认。
- 弹窗展示删除数量和前 5 个项目名称。
- 用户点击“取消”时抛出 `userCancelled`，不会删除任何文件。

### 1.6 Developer ID 公证检查
- `security find-identity -v -p codesigning` 仅发现 Apple Development 和 Apple Distribution 证书。
- 未发现 `Developer ID Application` 证书。
- `xcrun notarytool history` 返回 `Must provide credentials`。
- `security find-generic-password -s com.apple.gke.notary.tool` 未找到 notarytool 凭据。
- 当前环境无法真实完成 Developer ID 签名与公证，需要补齐证书和 App Store Connect/Apple ID 公证凭据。

## 4. 手工验证建议

1. 启动主 App。
2. 查看左侧栏，确认“评分”和“订购”不再展示。
3. 在 Finder 受监控目录中右键，确认动作按“新建 / 打开 / 工具”分组进入子菜单。
4. 点击任意子菜单叶子动作，确认仍能执行或转发给主 App。
5. 在“文件(夹)图标”页导入一张本地图片，确认表格预览、尺寸和右键图标设置生效。
6. 拖拽图标预设行，确认顺序保存并影响 Finder 右键菜单顺序。
7. 在 Finder 选中文件右键执行“剪切”，确认剪贴板中存在文件 URL，并在 Finder 中粘贴移动。
8. 执行“iShot”，确认本机安装 iShot 时能拉起应用；未安装时日志记录缺失。
9. 选中 PNG/JPEG 等图片执行“ICNS 转换”，确认同目录生成 `.icns` 文件。
10. 启用“彻底删除”后选择临时文件执行，确认弹出二次确认；点“取消”文件保留，点“彻底删除”文件被移除。
