# 开发验证记录 V3.9.4

日期：2026-06-09

## 目标

修复四个影响真实使用的问题：

1. 完全磁盘访问权限已在系统勾选，但 App 内仍显示未授权。
2. Finder 右键扩展只在桌面目录出现。
3. Finder 右键分组/子菜单需要确保新安装版本加载生效。
4. Sparkle 更新安装时需要降低旧主进程或旧 Finder 扩展残留的风险。

## 已完成

### 完全磁盘访问权限检测

- `FullDiskAccessPermission` 增加系统级 TCC 数据库 `/Library/Application Support/com.apple.TCC/TCC.db` 检测。
- 增加系统级和用户级 TCC 目录检测，避免 Mail、Messages、Safari 等目录不存在导致误判。
- 主 App Release entitlements 移除 `com.apple.security.app-sandbox`，避免沙盒阻断已经授予的完全磁盘访问能力探测。
- 配置界面原有 `onAppear`、App 激活、定时刷新逻辑继续保留，用户从系统设置返回后会自动刷新状态。

### Finder 扩展覆盖范围

- 默认监听目录从 `~/Desktop` 改为用户 Home 目录。
- 旧配置如果是早期默认的“仅桌面监听”，加载时自动迁移为 Home。
- 用户自定义的监听目录不会被迁移覆盖。
- Finder Extension 读取配置失败或配置目录不可用时，也回退到 Home，而不是 Desktop。

### Finder 右键二级分类

- 代码层已经按 `MenuGroup` 构建子菜单：`新建 / 打开 / 工具`。
- 如果本机仍看到扁平列表，优先判断为 `/Applications/右键增强.app` 或 Finder 扩展进程仍是旧版本，需要安装新构建并重启 Finder 扩展。

### Sparkle 更新安装清理

- 主 App 接入 `SPUUpdaterDelegate`。
- Sparkle 安装前、即将重启前、更新导致 App 终止时，主动终止 `RightClickFinderExtension` 运行进程。
- Sparkle 仍负责主 App 的退出、替换和重启；本次补强只处理 Finder 托管的旧 appex 进程残留。

## 手动验证建议

1. 构建并安装新版 App 到 `/Applications/右键增强.app`。
2. 退出旧 App，并结束旧 Finder 扩展进程或重启 Finder。
3. 打开 App，确认“通用设置 > 权限”在系统已勾选时显示为已授权。
4. 在 `~/Documents`、`~/Downloads`、任意 Home 子目录打开 Finder 右键菜单，确认扩展可出现。
5. 查看右键菜单是否出现 `新建 / 打开 / 工具` 二级分类。
6. 通过 Sparkle 更新安装时，观察日志中是否有 `Sparkle cleanup terminate Finder extension`。

## 已知限制

- macOS 没有公开 API 能直接读取完全磁盘访问权限状态；当前仍是能力探测，但探测范围已经扩大到 TCC 数据库。
- Developer ID 公证仍需要 Developer ID Application 证书和 `notarytool` 凭据，本机当前无法真实公证。
