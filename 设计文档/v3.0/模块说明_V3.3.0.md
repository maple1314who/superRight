## 2. Shared 模块

路径：
`SuperRight/Shared`

职责：
- 定义跨进程共享模型。
- 定义 App Group 常量。
- 定义配置读写 Store。
- 定义 IPC 请求模型。

关键文件：
- `Shared/Models/SharedConfiguration.swift`：共享配置根对象。
- `Shared/Models/AppSettings.swift`：全局开关。
- `Shared/Models/MenuItemConfiguration.swift`：基础菜单项配置。
- `Shared/Models/NewFileTemplateConfiguration.swift`：新建文件模板。
- `Shared/Models/FileDestinationConfiguration.swift`：目录型配置项。
- `Shared/IPC/AppExecutionRequest.swift`：Finder Extension 转主 App 的请求模型。
- `Shared/Storage/UserDefaultsConfigurationStore.swift`：App Group 配置读写。
- `Shared/Permissions/FullDiskAccessPermission.swift`：完全磁盘访问权限检测。

维护规则：
- 新增配置字段必须兼容旧 JSON。
- 新增默认菜单项必须在 `upgradedWithDefaults()` 中补齐。
- 新增 IPC 字段必须在 `AppExecutionRequest.init(from:)` 中给默认值。

## 3. ExtensionCore 模块

路径：
`SuperRight/Extension`

职责：
- 根据 Finder 上下文构建菜单。
- 判断菜单在空白处、文件、文件夹场景中的可见性。
- 提供可单测的动作执行器。

关键文件：
- `Extension/FinderSync/FinderSelectionContext.swift`：Finder 选中状态抽象。
- `Extension/MenuBuilder/MenuBuilder.swift`：菜单构建入口。
- `Extension/MenuBuilder/MenuDisplayItem.swift`：Finder 菜单展示模型。
- `Extension/ActionDispatcher/ActionDispatcher.swift`：动作执行参考实现。

维护规则：
- `MenuBuilder` 只构建菜单，不执行动作。
- 写文件、复制/移动、打开目录等动作优先由主 App 执行。
- 新增菜单来源时，在 `MenuBuilder.buildMenu()` 中合并并补单测。

## 4. AppCore 模块

路径：
`SuperRight/App`

职责：
- 主 App 配置界面。
- 配置 ViewModel。
- UI 表格、开关、路径配置和模块页面。

关键文件：
- `App/AppEntry/RootConfigurationView.swift`：配置界面根视图和主要模块 UI。
- `App/ViewModels/MenuManagementViewModel.swift`：配置读写入口。
- `App/Settings/ApplicationSettingsView.swift`：旧设置页组件，部分能力仍可复用。

维护规则：
- SwiftUI 视图不直接保存配置，统一调用 `MenuManagementViewModel`。
- 每个配置模块新增能力时，需要同步：
  1. Shared 模型
  2. ViewModel 写入方法
  3. UI 绑定
  4. MenuBuilder 或主 App 执行器
  5. 单元测试和验证文档

## 5. Finder Extension 模块

路径：
`右键增强/RightClickFinderExtension`

职责：
- 接收 Finder 右键菜单请求。
- 读取 Finder 当前目录和选中项。
- 从共享配置构建菜单。
- 将点击事件分发给 ExtensionCore 或转发给主 App。

关键文件：
- `RightClickFinderExtension/FinderSync.swift`

维护规则：
- `representedObject` 只保存 ObjC 桥接安全值。
- 复杂 payload 放在 `payloadByMenuTag`。
- 每次 `menu(for:)` 都刷新监听目录和配置。
- 需要主 App 执行的动作必须进入 `forwardToMainAppIfNeeded()`。

## 6. 主 App 执行模块

路径：
`右键增强/右键增强`

职责：
- 接收 Finder Extension 的 IPC 请求。
- 执行文件创建、复制/移动、打开目录、打开外部应用。
- 通过 App Group 队列兜底，避免请求丢失。

关键文件：
- `AppDelegate.swift`：App 生命周期、状态栏/Dock、权限引导。
- `AppExecutionRequestHandler.swift`：请求队列和动作执行。

维护规则：
- 新增 `AppExecutionAction` 后必须在 `execute(request:)` 中补分支。
- 执行失败必须写日志。
- 队列文件处理后必须删除，避免坏请求反复执行。
