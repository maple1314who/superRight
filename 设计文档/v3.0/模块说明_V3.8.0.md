## 2. Shared 模块

路径：
`SuperRight/Shared`

职责：
- 定义跨进程共享模型。
- 定义 App Group 常量。
- 定义配置读写 Store。
- 定义 IPC 请求模型。

关键文件：
- `Shared/Models/MenuActionType.swift`：右键菜单动作类型，包含 `permanentlyDelete`。
- `Shared/IPC/AppExecutionRequest.swift`：Finder Extension 转主 App 的请求模型，包含 `permanentlyDelete`。
- `Shared/Models/ToolboxItemConfiguration.swift`：工具箱功能项，默认项包含关闭状态的“彻底删除”。
- `Shared/Models/SharedConfiguration.swift`：旧配置升级时自动追加缺失默认工具箱项。

维护规则：
- 新增配置字段必须兼容旧 JSON。
- 新增默认工具箱项必须通过 `upgradedWithDefaults()` 迁移补齐。
- 破坏性工具箱动作默认关闭，避免升级后直接暴露危险入口。
- 新增 IPC 动作必须同步 Finder Extension 转发和主 App 执行分支。

## 3. ExtensionCore 模块

路径：
`SuperRight/Extension`

职责：
- 根据 Finder 上下文构建菜单。
- 判断菜单在空白处、文件、文件夹场景中的可见性。
- 提供可单测的动作执行器。

关键文件：
- `Extension/MenuBuilder/MenuBuilder.swift`：工具箱菜单构建入口；“彻底删除”启用后只在选中文件或文件夹时可见。
- `Extension/ActionDispatcher/ActionDispatcher.swift`：对“彻底删除”保持 forwarded-only，实际执行由主 App 完成。

维护规则：
- `MenuBuilder` 只构建菜单，不执行动作。
- 涉及系统 UI、权限或破坏性文件操作的动作优先转发主 App。
- 新增菜单来源时同步单元测试。

## 4. AppCore 模块

路径：
`SuperRight/App`

职责：
- 主 App 配置界面。
- 配置 ViewModel。
- UI 表格、开关、路径配置和模块页面。

关键文件：
- `App/AppEntry/RootConfigurationView.swift`：配置界面根视图，工具箱页面自动展示默认项迁移后的“彻底删除”。
- `App/ViewModels/MenuManagementViewModel.swift`：配置读写入口。

维护规则：
- SwiftUI 视图不直接保存配置，统一调用 `MenuManagementViewModel`。
- 每个配置模块新增能力时，需要同步 Shared 模型、ViewModel 写入方法、UI 绑定、MenuBuilder 或主 App 执行器、单元测试和验证文档。

## 5. Finder Extension 模块

路径：
`右键增强/RightClickFinderExtension`

职责：
- 接收 Finder 右键菜单请求。
- 读取 Finder 当前目录和选中项。
- 从共享配置构建菜单。
- 将点击事件分发给 ExtensionCore 或转发给主 App。

关键文件：
- `RightClickFinderExtension/FinderSync.swift`：将 `permanentlyDelete` 映射为主 App 执行请求，并携带 `sourcePaths`。

维护规则：
- `representedObject` 只保存 ObjC 桥接安全值。
- 复杂 payload 放在 `payloadByMenuTag`。
- 每次 `menu(for:)` 都刷新监听目录和配置。
- 需要主 App 执行的动作必须进入 `forwardToMainAppIfNeeded()`。

## 6. 主 App 执行模块

路径：
`右键增强/右键增强`

职责：
- 接收 Finder Extension 的 IPC 请求。
- 执行文件创建、复制/移动、打开目录、打开外部应用、写入或清除 Finder 自定义图标、工具箱文件操作。
- 通过 App Group 队列兜底，避免请求丢失。

关键文件：
- `AppExecutionRequestHandler.swift`：`permanentlyDelete(for:)` 直接删除 Finder 选中项，不经过废纸篓。

维护规则：
- 新增 `AppExecutionAction` 后必须在 `execute(request:)` 中补分支。
- 执行失败必须写日志。
- 队列文件处理后必须删除，避免坏请求反复执行。
- 破坏性动作必须保留清晰注释和默认关闭策略。
