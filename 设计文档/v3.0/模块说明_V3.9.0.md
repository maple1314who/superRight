## 2. 数据流

1. 主 App 启动后 Sparkle 根据 `SUFeedURL` 拉取 `appcast.xml`。
2. Sparkle 读取 Appcast 中的版本、下载地址和 EdDSA 签名。
3. Sparkle 下载 GitHub Release 中的 DMG 包。
4. Sparkle 使用 `SUPublicEDKey` 校验 DMG 签名。
5. 校验通过后由 Sparkle installer launcher 执行替换安装。

## 3. 安全规则

- 只提交 Sparkle 公钥，不提交私钥或 `.sparkle/` 目录。
- DMG 包必须由 Sparkle 工具生成签名后再发布。
- 修改 `SUFeedURL` 或公钥后必须重新构建主 App。
- 主 App 保留 sandbox，Sparkle 仅通过明确 entitlements 获取所需 XPC 能力。
