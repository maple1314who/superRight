## 4. 发版规则

- 功能版本：递增 `MAJOR.MINOR.0`，例如 `3.9.0`。
- 修复版本：递增 `MAJOR.MINOR.PATCH`，例如 `3.9.1`。
- 构建号：使用日期加序号，格式为 `yyyyMMddNN`，例如 `2026060909`。
- 每次发版必须同步更新：
  - Xcode 工程 `MARKETING_VERSION`
  - Xcode 工程 `CURRENT_PROJECT_VERSION`
  - `CHANGELOG.md`
  - 对应开发验证记录
  - Sparkle Appcast 和 GitHub Release 资产
