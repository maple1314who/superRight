# 验证操作文档 V3.0.0

## 1. 文档目的
本文档用于验证右键增强工具 V3.0.0 是否满足真实使用要求。
重点不再只是代码层，而是：
- 真实 Finder GUI 菜单
- 真实点击效果
- 多目录监听
- 主 App 配置联动
- 长期自用可行性

## 2. 验证前提
开始前请确认：
1. 主工程路径：
   `/Users/maple/Documents/Project/Swift/右键增强`
2. Swift Package 路径：
   `/Users/maple/Documents/Project/Swift/SuperRight`
3. 主 App 与 Finder Extension 可以编译
4. Finder Extension 已能运行
5. 主 App 已能打开
6. 至少已配置一个监听目录

## 3. 启动准备
### 3.1 启动主 App
- 在 Xcode 中运行 `右键增强`
- 确认主 App 配置界面正常打开

### 3.2 启动 Finder Extension
- 在 Xcode 中运行 `RightClickFinderExtension`
- 如有需要附着 Finder，则选择 Finder
- 如必要执行：`killall Finder`

### 3.3 观察日志
建议同时观察：
- Finder Extension 日志
- 主 App 执行日志

若项目已落盘日志文件，也建议同时使用 `tail -f` 观察。

## 4. 验证范围
V3 重点验证以下动作：
1. 新建文件夹
2. 在终端中打开
3. 在 iTerm 中打开
4. 用 VS Code 打开
5. 复制路径

以及：
6. 多目录监听
7. 配置联动
8. 长期运行方式

## 5. 配置联动验证
### 用例 A1：菜单开关生效
步骤：
1. 打开主 App
2. 关闭“在终端中打开”
3. 回到 Finder 在监听目录右键
4. 再打开“在终端中打开”
5. 再次回到 Finder 右键

预期结果：
- 关闭后菜单中不显示“在终端中打开”
- 开启后菜单中恢复显示

### 用例 A2：复制路径开关生效
步骤：
1. 在主 App 中关闭“复制路径”
2. Finder 再次右键
3. 开启“复制路径”
4. Finder 再次右键

预期结果：
- 菜单显示与配置一致

### 用例 A3：排序生效
步骤：
1. 在主 App 中调整菜单排序
2. 回到 Finder 重新右键

预期结果：
- 菜单顺序与配置一致

## 8. 主 App 执行动作验证
### 用例 D1：主 App 运行时新建文件夹
步骤：
1. 保持主 App 运行
2. Finder 点击“新建文件夹”

预期结果：
- 目录创建成功
- 主 App 日志中可看到 request received / createFolder success

### 用例 D2：主 App 未运行时新建文件夹
步骤：
1. 关闭主 App
2. Finder 点击“新建文件夹”
3. 观察是否自动拉起主 App
4. 或再次手动打开主 App

预期结果：
- 若支持自动拉起，则目录自动创建
- 若自动拉起失败，但请求留在队列中，则手动启动主 App 后可补偿执行
- 行为必须与文档一致

## 11. 失败排查建议
### 11.1 菜单能显示但点击无效
检查：
- `handleMenuItem`
- payload 解码
- dispatcher.execute
- request forward
- main app request received

### 11.2 菜单跟配置不同步
检查：
- App Group
- `UserDefaultsConfigurationStore`
- suiteName / key
- 主 App / Extension 日志中的 menuItems 快照

### 11.3 新建文件夹失败
检查：
- 是否由主 App 执行
- 主 App 是否运行
- 队列是否落盘
- 主 App 是否消费请求

### 11.4 打开 Terminal / iTerm / VS Code 失败
检查：
- 应用路径
- open 命令参数
- 是否安装目标应用
- ActionDispatcher 日志

## 13. V3 验收标准
满足以下条件可视为 V3.0.0 基本完成：
1. 新建文件夹真实可用
2. 在终端中打开真实可用
3. 复制路径真实可用
4. iTerm / VS Code 已安装时真实可用，未安装时按配置隐藏或明确不适用
5. 配置联动稳定
6. 支持多个监听目录
7. 主 App 与 Finder Extension 的职责边界清晰
8. 有明确的长期自用运行说明
