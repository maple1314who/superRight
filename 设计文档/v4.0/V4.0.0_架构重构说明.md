# V4.0.0 架构重构说明

日期：2026-06-09

## 目标

把 Finder Extension 侧菜单执行链从集中式 `switch` 重构为可扩展的 V4 架构。后续新增菜单、系统版本差异实现、日志/埋点/重试能力时，不再继续扩大单个执行器文件。

## 设计模式落地

### 策略模式

- 协议：`MenuActionStrategy`
- 入口：`ActionDispatcher.execute(...)`
- 已拆分策略：
  - `CreateFolderActionStrategy`
  - `CreateFileActionStrategy`
  - `OpenExternalApplicationActionStrategy`
  - `CopyPathActionStrategy`
  - `TransferItemsActionStrategy`
  - `OpenDirectoryActionStrategy`
  - `ForwardedOnlyActionStrategy`

每个策略只处理一类动作流程，不直接创建 `Process`、写剪贴板或判断系统版本。

### 工厂模式

- 类型：`MenuActionStrategyFactory`
- 职责：维护 `MenuActionType -> MenuActionStrategy` 映射。
- 新增菜单动作时优先新增策略，然后在 `defaultStrategies()` 注册。

这样执行入口不用再写大 `switch`，菜单动作扩展点集中、可测试。

### 适配器模式

- 文件系统适配器：`FileSystemActionAdapter`
- 外部 App 适配器：`ExternalApplicationActionAdapter`
- 系统版本提供者：`SystemVersionProviding`

适配器隔离 macOS API 差异。当前外部 App 打开仍统一使用 `/usr/bin/open`，但已经把系统版本注入到适配器中；如果后续某些 macOS 版本需要切换 `NSWorkspace`、LaunchServices 或 App 专用 URL Scheme，只需要替换适配器实现，不需要改策略。

### 观察者模式

- 协议：`ActionExecutionObserving`
- 默认实现：`NSLogActionExecutionObserver`
- 组合实现：`CompositeActionExecutionObserver`

观察者记录动作开始、成功和失败。后续可以接入调试窗口、文件日志、统计埋点或失败重试，不污染策略代码。

### 责任链模式

- 基类：`ActionExecutionPreflightLink`
- 默认链：`ActionExecutionPreflightLink.defaultChain()`
- 已接入节点：
  - `DefaultFileNamePreflightLink`
  - `DestinationPathPreflightLink`
  - `SelectedItemsPreflightLink`

责任链在进入具体策略前执行，用于统一校验动作前置条件。顺序是有业务含义的：先检查配置完整性，再检查 Finder 运行时输入，保证错误优先指向根因。

### 空对象模式

- 类型：`ForwardedOnlyActionStrategy`

Finder Extension 不能安全直接执行的动作统一映射到空对象策略，策略返回 `nil` 并记录日志，由 `FinderSync` 的转发链交给主 App 执行。这避免了执行入口出现大量“什么都不做但必须存在”的特殊分支。

## 模块边界

- `ActionDispatcher`：执行门面，只负责组装上下文、触发观察者、调用工厂选择策略。
- `ActionExecutionRequestContext`：策略只读上下文，封装菜单项、Finder 选择和配置。
- `ActionExecutionAdapters`：策略依赖的能力集合。
- `MenuActionStrategyFactory`：策略注册和选择中心。
- `ActionAdapters`：平台和系统 API 隔离层。
- `ActionExecutionPreflightChain`：策略执行前的责任链校验层。

### 调整范围

- `SharedConfiguration` 对旧配置补齐默认主菜单展示偏好，避免历史配置缺少 `promotedToMainMenu` 时“复制路径”仍留在子菜单。

- Finder 根菜单只在高频项和后续分类子菜单之间保留一个分隔线，不再对每个分类子菜单都插入分隔，减少菜单视觉间隙。
